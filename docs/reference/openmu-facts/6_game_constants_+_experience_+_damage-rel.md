### experience curve (character level)
- Formula (identical string in initializer and hardcoded default in game logic): exp(level) = 0 if level==0; 10*(level+8)*(level-1)^2 if level<256; 10*(level+8)*(level-1)^2 + 1000*(level-247)*(level-256)^2 if level>=256
- Semantics: exp(level) is the TOTAL accumulated experience required to reach that level (Player levels up when total experience >= table[currentLevel+1])
- Table shape: precomputed long[] with (MaximumLevel + 2) entries, index = level, index 0..MaximumLevel+1; for MaximumLevel=400 that is 402 entries; generated by evaluating the formula per level (GameContext.CreateExpTable)
- Sample values: level 1 = 0; level 2 = 100; level 10 = 14580; level 255 = 1697560640; break at level 256 adds the second cubic term
- Per-kill base experience formula (AttackableExtensions.CalculateBaseExperience): base = (targetLevel+25)*targetLevel/3; if killerLevel > targetLevel+10 then base *= (targetLevel+10)/killerLevel; if targetLevel >= 65 then base += (targetLevel-64)*(targetLevel/4); result = max(base,0) * 1.25; summoned monsters yield 0
- Awarded exp is randomized: multiplied by uniform random in [RandomExperienceMinMultiplier, RandomExperienceMaxMultiplier] = [0.8, 1.2], then by ExperienceRate (and per-map/monster factors owned by other areas)

### global base attribute values (apply to all characters)
- MoneyAmountRate: constant 1.0 (multiplier on dropped money amount)
- RandomExperienceMinMultiplier: constant 0.8
- RandomExperienceMaxMultiplier: constant 1.2
- MovementSpeedFactor: constant 1.0
- All four set in GameConfigurationInitializerBase for every version; VersionSeasonSix additionally adds MaximumAllianceSize = 5 (guild alliance, flag: alliances likely pre-S3 but only S6 dataset sets it)

### default drop item groups (global)
- DropItemGroup fields used: Chance (double, probability 0..1), ItemType (SpecialItemType enum), ItemLevel (byte, optional), Description (string)
- Money group: chance 0.5, type Money — all versions
- Random item group: chance 0.3, type RandomItem — all versions
- Excellent item group: chance 0.0001, type Excellent, ItemLevel 0 — only created when the version has the Excellent option type: absent in Version075, present in Version095d and VersionSeasonSix
- Jewels group: chance 0.001, type Jewel — all versions
- These four are registered as default drop groups for every map

### default item option definitions (created for all versions)
- Luck option: add chance 0.25, max 1 per item, effect = CriticalDamageChance +0.05 (5%)
- Generic '+option' definitions created via CreateOptionDefinition: add chance 0.25, adds randomly, max 1 per item, option levels 1..MaximumOptionLevel (=4), value per level = level * baseValue
- DefenseBase option: baseValue 4 (so +4/+8/+12/+16)
- PhysicalBaseDmg option: baseValue 4
- WizardryBaseDmg option: baseValue 4
- DefenseRatePvm option: baseValue 5 (so +5/+10/+15/+20)
- MaximumOptionLevel: 4 (InitializerBase default and Version075/095d Constants); exception: Version075 Jewelery uses 3
- MaximumItemLevel: 11 in Version075 and Version095d (Constants.cs); Season Six uses item levels up to 15
- VersionSeasonSix additionally creates: BaseDamageBonus option (phys+wiz combined) and CurseBaseDmg option (post-S3, Summoner)

### level-dependent damage table
- Shape: list of {Level: int, Damage: int} pairs (/tmp/openmu-ref/src/DataModel/Configuration/LevelDependentDamage.cs)
- Semantics per docs: a damage value belonging to a level (intended for skills/effects whose damage scales stepwise with level)
- IMPORTANT: defined in the data model but referenced NOWHERE else in the codebase — no initializer populates it and no game logic reads it; it is dead data in current OpenMU

### hit outcome
- HitInfo record: HealthDamage (uint), ShieldDamage (uint), Attributes (DamageAttributes flags), ManaToll (uint, default 0; mana drained from defender e.g. Soul Barrier)
- Miss is represented as HitInfo(0, 0, Undefined) when the hit-chance roll fails
- Hit-chance reference data: minimum hit chance constant 0.03 (3%); otherwise chance = 1 - defenseRate/attackRate when attackRate > defenseRate; PvP uses AttackRatePvp/DefenseRatePvp, PvM uses AttackRatePvm/DefenseRatePvm
- 'Overrate' rule: if defender DefenseRatePvm > attacker AttackRatePvm (PvM only), damage is scaled by 0.3
- Minimum damage floor: max(1, attackerLevel / 10)
- Critical hit = deal max damage; Excellent hit = 1.2 * max damage (+ bonuses); both rolled from CriticalDamageChance / ExcellentDamageChance
- Shield split (S3+ SD system): default 90% of damage to shield, 10% to health; ratio adjusted by attacker ShieldDecreaseRateIncrease and defender ShieldRateIncrease, clamped 0..1; ShieldBypassChance skips shield entirely
- Elemental effect application: applies with probability 1/(resistance+1); resistance >= 255 means immune; a successful lightning-element application also relocates the target by 1 tile randomly

### damage calculation attribute inputs (cross-check list for stats schema)
- Attacker base damage: MinimumPhysBaseDmg, MaximumPhysBaseDmg, MinimumWizBaseDmg, MaximumWizBaseDmg, WizardryAttackDamageIncrease; post-S3: MinimumCurseBaseDmg, MaximumCurseBaseDmg, CurseAttackDamageIncrease, FenrirBaseDmg
- Attacker chance stats: CriticalDamageChance, ExcellentDamageChance, DefenseIgnoreChance, DoubleDamageChance; hit chance: AttackRatePvm, AttackRatePvp
- Attacker bonus stats: CriticalDamageBonus, ExcellentDamageBonus, SkillDamageBonus, SkillMultiplier, AttackDamageIncrease, ComboBonus, GreaterDamageBonus, FinalDamageBonus, FinalDamageIncreasePvp (PvP only)
- Attacker equipment flags read: HasDoubleWield (110% dmg: dmg doubled after halving skill dmg), IsTwoHandedWeaponEquipped + TwoHandedWeaponDamageIncrease, IsDinorantEquipped (x1.3 on skill-less attack), IsBowEquipped/IsCrossBowEquipped (ammo), AmmunitionConsumptionRate, AmmunitionAmount
- Skill-scoped attributes (per-skill attribute system): SkillLevel, SkillBaseDamageBonus, SkillBaseMultiplier, SkillFinalDamageBonus, SkillFinalMultiplier, SkillExtraManaCost; skill cost reductions: ManaUsageReduction, AbilityUsageReduction against CurrentMana/CurrentAbility

- Attacker level for damage floor: TotalLevel (player) or Level (monster)
- Defender stats: DefensePvm, DefensePvp, DefenseDecrement (multiplier applied to defense), DefenseRatePvm, DefenseRatePvp, ArmorDamageDecrease, DamageReceiveDecrement, Level (magic-effect duration reduction)

### elemental types
- Enum in /tmp/openmu-ref/src/Persistence/Initialization/ElementalType.cs: Undefined, Ice, Poison, Lightning, Fire, Earth, Wind, Water (8 values, ordinal 0..7)
- Each element maps to a defender resistance attribute (e.g. Ice -> IceResistance) used as the skill's ElementalModifierTarget
- Each resistance maps 1:1 to an attacker damage-bonus attribute (Ice -> IceDamageBonus etc.) added to skill min/max damage
- Elemental magic effect SubType is encoded as 0xFF - elementalType ordinal
- Standard elemental effect durations from initializers: Ice/Iced 10s (Ice Arrow Freeze 5s), Poison 20s for skill Poison / 10s for Decay, Iced movement speed factor 0.5 (multiplicative); Lightning element causes 1-tile random displacement
- Version075 skills use elements Ice/Poison/Lightning/Fire (Earth/Wind/Water appear with later skills)

## Version notes
- All GameConfiguration scalars (max level 400, exp rate 1.0, exp formula, drop duration 60s, durability constants, etc.) come from the shared GameConfigurationInitializerBase; Version075, Version095d and VersionSeasonSix do NOT override any of them.
- Item option types: 0.75 has only Option + Luck; 0.95d adds Excellent (and ExcellentOptions initializer); S6 takes every defined option type (Harmony, Guardian, Ancient/set, Socket, Wing, etc.).
- Excellent drop group (chance 0.0001) exists only where Excellent option type exists: 0.95d and S6, not 0.75.
- Max item level constant: 11 in 0.75 and 0.95d datasets; S6 content uses up to 15. Max option level 4 everywhere (0.75 jewelry: 3). Max droppable option level: 3 (all versions).
- Jewel mixes (packed jewels) are only initialized in S6 (10 mixes); 0.75/0.95d have none.
- S6-only initializer content in game config: SocketSystem, HarmonyOptions, GuardianOptions, AncientSets, PackedJewels, Misc, quests, BloodCastle/ChaosCastle mini-game definitions, MaximumAllianceSize global value, BaseDamageBonus and CurseBaseDmg item options; 0.95d adds over 0.75: EventTicketItems, BoxOfLuck, invasion mobs, DevilSquare definition.
- Inventory extension slots (128) and the S6 store offset are Season-6 layout; pre-S3 has only 12 equip + 64 inventory before the store.
- Elemental types Earth/Wind/Water are defined for all versions but 0.75 skills only use Ice/Poison/Lightning/Fire.

## Post-S3 exclusions
- Master level system: MaximumMasterLevel (200), MasterExperienceRate, MasterExperienceFormula '(505*l^3)+(228045*l^2)+(35278500*l)', MinimumMonsterLevelForMasterExperience (95), MasterSkillRoots collection (S4+)
- Shield/SD system: HitInfo.ShieldDamage split (90/10 default), MaximumShield/CurrentShield, ShieldBypassChance, ShieldDecreaseRateIncrease, ShieldRateIncrease, DefenseDecrement usage tied to it (S3+); for pre-S3 all damage goes to health
- DamageType Curse, SummonedMonster, Fenrir; Berserker*/WizardryAndCurse* Summoner stats (S6)
- DamageAttributes RageFighterStreakHit / RageFighterStreakFinalHit (Rage Fighter, S6)
- Master-skill-tree weapon bonus damage stats (Bow/CrossBow/TwoHandedSword/Spear/Scepter/Glove/OneHandedSword/Mace/Stick/Staff bonuses, MasterSkillPhysBonusDmg) consumed by the damage calc (S4+)
- HitInfo.ManaToll driven by SoulBarrierManaTollPerHit (S6 Soul Barrier rework); classic Soul Barrier is a plain damage reduction
- SpecialItemType.SocketItem, socket system, harmony/guardian options, packed jewels, jewel mixes beyond basic jewels
- Inventory extensions (4 x 4 rows = 128 slots) and S6 store slot offset
- RavenBonusDamage / AttackerSurrogate (Dark Lord raven) — not present in 0.75/0.95d datasets
- FinalDamageIncreasePvp (PvP wing/3rd-wing stat) and TwoHandedWeaponDamageIncrease as used by S6 damage calc — verify era before including
- Fenrir/horse movement speeds (17/19) and UpgradedFenrir

## Open questions
- LevelDependentDamage exists in the data model but is populated and read nowhere in OpenMU — include a {level:int, damage:int} table in the mu-core schema anyway, or drop it?
- OpenMU sets MaximumLevel=400 and the modern two-piece exp formula for ALL versions including 0.75/0.95d; historically 0.75 had a lower cap and simpler curve. Accept OpenMU's 400 + formula for the pre-S3 target, or set a version-accurate cap?
- Per-kill exp formula (incl. the >=65 bonus term and the flat *1.25) is OpenMU's single implementation for all versions — accept as pre-S3 behavior?
- DamageAttributes.Combo / ComboBonus (DK combo, introduced ~1.0) and DamageAttributes.Triple: inside or outside the pre-S3 target's version cut?
- Hardcoded logic constants not in config: min hit chance 0.03, overrate damage factor 0.3, min damage floor level/10, classic-duel damage factor 0.6, Dinorant x1.3, double wield x2, Chaos Castle PvP halving — should mu-core expose these as configurable constants or hardcode them?
- Excellent damage stats (ExcellentDamageChance/Bonus) enter via excellent options which OpenMU gives 0.95d+ — confirm excellent items are in scope for the pre-S3 target.
- MaximumInventoryMoney/MaximumVaultMoney = int.MaxValue in OpenMU; classic servers capped zen at 2,000,000,000 — pick a cap.
- GreaterDamageBonus is consumed by the damage calc but is fed by Jewel-of-Harmony-era options in S6 data — keep the stat in the schema for pre-S3 or exclude with harmony?
- Elemental ailment magic-effect durations (Iced 10s, Freeze 5s, Poison 20s/10s, iced speed factor 0.5) live in skill initializers — treat as fixed constants or per-skill data in mu-core?
